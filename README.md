# BCM02 JavaScript 自動化腳本

這是一個用於自動化執行 BCM02 報表填寫流程的 JavaScript 腳本。旨在簡化重複性的表單填寫與點擊操作，提升工作效率。

## 功能簡介

本腳本主要用於自動化處理客戶互動紀錄的填寫與簽核流程，包含：
1.  自動搜尋「本月未檢視戶數」。
2.  循環處理客戶列表。
3.  自動點擊「更多」並新增互動紀錄。
4.  自動填寫表單內容（客戶來源、聯繫管道、聯繫結果等）。
5.  自動送出並關閉視窗，接續處理下一位客戶。

## 執行方式

1.  複製 `BCM02_automation.js` 的完整程式碼。
2.  在瀏覽器中打開要自動化執行的報表頁面。
3.  打開開發者主控台 (Console)：
    *   快速鍵：`F12` 或 `Ctrl` + `Shift` + `I`
    *   或點選瀏覽器選單中的「更多工具」->「開發人員工具」，然後切換到 "Console" 分頁。
4.  在 Console 中貼上程式碼 (`Ctrl` + `V`)，然後按 `Enter` 執行。

## 疑難排解

### 出現「已在偵錯工具中暫停」(Paused in debugger)

如果在執行過程中程式停住，並顯示「已在偵錯工具中暫停」，通常是因為 Chrome 開發者工具開啟了 **「發生例外狀況時暫停」** 的功能。這會導致網頁原本的程式碼（非本腳本）拋出任何錯誤時，瀏覽器強制中斷執行。

**解決方法：**

1.  在開發者工具中，找到「停用中斷點」按鈕（圖示通常是一個被劃掉的六角形或類似標誌），或按快速鍵 `Ctrl` + `F8`。
2.  接著按「繼續執行指令碼」按鈕（播放鍵圖示），或按快速鍵 `F8`。
3.  腳本應會繼續執行。

> 建議在執行腳本前，先確認開發者工具的「發生例外狀況時暫停」功能已關閉 (圖示應為灰色/未啟用狀態)。（乙仔：這句話我看不懂，都是發生中斷時才去按「Ctrl + F8」和「F8」）

乙仔：白話點再講一遍，就是在執行腳本時，出現「已在偵錯工具中暫停」的提示，這時候要按「Ctrl + F8」和「F8」，才能讓腳本繼續執行。把「來源模式(偵錯模式)(小蟲按鈕)」的第6個選項圓球點成藍色（或按快速鍵「Ctrl+F8」停用中斷點）。

## 開發提示詞

僅供參考，建議不要一次全部丟給AI開發，通常AI一次開發的不能用，分段開發完成的時間，會比AI一次開發完再除錯的時間，來得更有效率。

請寫一個自動化簽核JavaScript
１BCM02-1：看「本月未檢視戶數」欄位，是否有不是0的數字，如有，則點選進入BCM02-2
２BCM02-2「淨值管理效益定期檢視客戶明細表」：點選BCM02-2的「客戶姓名」欄位的第一個客戶
３BCM02-3「客戶首頁」：接著點選BCM02-3「客戶首頁」的「互動紀錄」的「更多>>」按鈕
４BCM02-4「互動紀錄查詢」：點選「新增」。
５接著在BCM02-5「互動記錄新增」的每一個欄位都選第一項如下：
客戶來源：既有客戶
聯繫管道：資料檢視
聯繫結果：資料檢視
聯繫花費時間：0~5分鐘
請選擇「定期檢視淨值管理效益，持續關心客戶。」，然後再按「>」按鈕，將「定期檢視淨值管理效益，持續關心客戶。」移到右邊欄位。最後點選「確定」。
接著關閉BCM02-4「互動紀錄查詢」，接著關閉BCM02-3「客戶首頁」。回到步驟２BCM02-2「淨值管理效益定期檢視客戶明細表」：點選BCM02-2的「客戶姓名」欄位的第一個客戶，如BCM02-2表單空白已無客戶，則按右上角X關閉BCM02-2，回到BCM02-1，看看「本月未檢視戶數」欄位，是否有不是0的數字，如有，則點選進入BCM02-2，如無，則顯示「自動化簽核報表已完成！」
